# Pattern: 基于证据的论证

## 适用场景

适用于所有需要建立可信论点的任务：
- 研究报告
- 商业提案
- 技术决策文档
- 竞争分析

## 约束规则

### 核心原则

**每个论点必须有可验证的证据支撑**

证据类型（按可信度排序）：
1. 同行评审的学术文献
2. 官方统计数据
3. 实验/测试结果
4. 行业报告
5. 案例研究
6. 专家访谈

### 必须遵守

1. **论点-证据绑定**
   - 每提出一个论点，立即提供证据
   - 证据必须直接支持论点（非间接推测）
   - 明确标注证据来源

2. **证据质量标准**
   - 优先使用原始数据源（非转述）
   - 数据必须时效性（默认：3 年内）
   - 避免单一来源（关键论点需 2+ 来源）

3. **透明度要求**
   - 承认证据的局限性
   - 说明样本大小、时间范围等关键参数
   - 对冲突证据进行讨论

4. **引用完整性**
   - 格式：`论点内容（作者/机构，年份：具体页码或章节）`
   - 数据引用需包含：数值 + 单位 + 来源 + 采集时间

### 禁止

- ❌ 未经验证的推测性陈述
- ❌ "众所周知"、"显而易见"等无证据的权威表达
- ❌ 选择性引用（只引用支持己方的证据）
- ❌ 过时数据（无说明情况下使用 5 年前的数据）
- ❌ 模糊量词（"许多"、"大部分"）替代具体数据
- ❌ 因果混淆（相关性 ≠ 因果性）

### 证据强度分级

使用不同表达区分证据强度：

| 强度 | 表达方式 | 适用情况 |
|------|---------|---------|
| **强证据** | "研究表明"、"数据显示" | 多个高质量来源一致 |
| **中等证据** | "研究发现"、"调查显示" | 单一可信来源 |
| **弱证据** | "有研究指出"、"初步证据表明" | 样本有限或方法存疑 |
| **推测** | "可能"、"或许"、"理论上" | 基于逻辑推理，无直接证据 |

## 输出规范

### 标准论证结构

```markdown
**论点**：[清晰的主张]

**证据**：
- [证据 1]（来源，年份）
- [证据 2]（来源，年份）

**分析**：[解释证据如何支持论点]

**局限**：[承认证据的边界条件]
```

### 数据呈现规范

**定量数据**：
```
错误：性能有显著提升
正确：性能提升 34%（n=500, p<0.01）（Zhang et al., 2023）
```

**定性发现**：
```
错误：用户普遍认为
正确：72% 的受访用户（n=150）表示满意（UserVoice 调研，2024.Q1）
```

**趋势描述**：
```
错误：市场份额持续增长
正确：市场份额从 2020 年的 15% 增长至 2023 年的 28%（Gartner，2024）
```

### 引用标注

**学术文献**：
```
多智能体系统的涌现行为在临界规模后显著增强（Bonabeau, 2002: 287-289）。
```

**统计数据**：
```
2023 年全球 AI 投资达 1542 亿美元（IDC，2024.03）。
```

**实验结果**：
```
A/B 测试显示，新算法的响应时间减少 42%（内部测试，2024.01.15，n=10000）。
```

## 检查清单

### 执行前

- [ ] 确认可访问足够的证据来源（文献库、数据库）
- [ ] 明确论证的可信度要求（学术 vs 商业）
- [ ] 了解目标受众对证据类型的偏好

### 执行中

- [ ] 每写一个论点，立即搜索支撑证据
- [ ] 遇到冲突证据时，记录并后续讨论
- [ ] 对关键数据进行交叉验证（2+ 来源）

### 执行后

- [ ] 通读全文，标记所有未引用的论断
- [ ] 验证每个数据的来源可追溯
- [ ] 检查是否存在过度推测
- [ ] 确认引用格式统一
- [ ] 评估证据的时效性（是否需要更新）

## 示例对比

### ❌ 缺乏证据

```
多智能体系统在工业界得到了广泛应用。许多公司都在使用
这项技术，效果非常好。专家普遍认为这是未来的发展方向。

显然，该技术将继续快速发展。
```

**问题**：
- "广泛应用" — 无具体数据
- "许多公司" — 模糊量词
- "非常好" — 主观评价，无指标
- "专家普遍认为" — 未引用具体专家
- "显然" — 无证据的断言
- "将继续" — 预测无依据

### ✅ 基于证据

```
多智能体系统的工业应用呈现增长态势。Gartner（2023）的
调研显示，2022 年采用 MAS 的企业占比为 23%，较 2020 年
的 12% 增长近一倍。在物流优化领域，DHL 的实证案例表明，
MAS 使路径规划效率提升 31%（DHL Innovation Report, 2023: 45）。

McKinsey（2024）的技术趋势报告将 MAS 列为"未来 5 年企业
重点投资的 10 项技术"之一，预测年复合增长率达 18%。然而，
该预测基于对 500 家大型企业的调研，中小企业的采纳情况
仍需进一步观察。
```

**优点**：
- 具体数据：23% vs 12%
- 明确来源：Gartner, DHL, McKinsey
- 时间标注：2020, 2022, 2023, 2024
- 案例具体：DHL, 31%
- 承认局限：中小企业数据不足

## 特殊场景处理

### 缺乏直接证据时

**策略 1：使用类比证据**
```
虽然 MAS 在制造业的数据有限，但其在物流领域的成功
（效率提升 30%，Amazon, 2022）提示了相似场景的潜力。
```

**策略 2：理论推导**
```
基于复杂系统理论（Holland, 1995），当智能体数量超过
临界值时，理论上应出现涌现行为。虽然实证数据尚不充分，
但初步实验（n=3）支持该假设。
```

**策略 3：明确标注推测**
```
目前尚无直接证据表明 X。但基于 Y 和 Z 的观察，我们推测
可能存在这种关联。这一假设需要进一步验证。
```

### 处理冲突证据

```
关于 MAS 的能耗问题，研究结论存在分歧。Smith 等（2022）
报告能耗增加 15%，而 Wang 等（2023）则观察到降低 8%。
差异可能源于：
1. 样本规模不同（Smith: n=50, Wang: n=200）
2. 场景差异（Smith 聚焦实时系统，Wang 关注批处理）

综合两项研究，可初步认为：MAS 的能耗表现与应用场景
高度相关，需具体问题具体分析。
```

### 引用二手来源

```
优先：直接引用原始研究
次选：明确标注二手来源

错误：根据 Zhang（2023）的研究...
正确：根据 Smith（2020）的研究（转引自 Zhang, 2023: 112）...
```

## 质量评估标准

生成内容应满足：

- **可验证性**：读者能追溯到原始证据源
- **充分性**：每个关键论点至少 1 个证据
- **相关性**：证据直接支持论点（无逻辑跳跃）
- **时效性**：数据在有效期内
- **平衡性**：讨论不同观点，不隐藏矛盾证据
