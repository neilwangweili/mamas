# Pattern: 学术写作规范

## 适用场景

适用于所有需要产出学术性内容的任务：
- 论文撰写
- 文献综述
- 研究报告
- 技术白皮书

## 约束规则

### 必须遵守

1. **结构完整性**
   - 包含：标题、摘要、关键词、引言、主体、结论、参考文献
   - 主体部分使用分级标题（不超过 4 级）
   - 每个章节有明确的主题句

2. **语言规范**
   - 使用第三人称客观叙述
   - 避免主观情绪化表达
   - 术语使用一致（首次出现时定义）
   - 中文学术用语标准化

3. **论证逻辑**
   - 论点 → 论据 → 论证 → 结论 的完整链条
   - 避免逻辑跳跃
   - 反驳观点时需公正呈现

4. **引用规范**
   - 所有非原创观点必须引用
   - 引用格式统一（默认：作者-年份制）
   - 参考文献按首字母排序

### 禁止

- ❌ 口语化表达（"其实"、"比较"、"非常"等）
- ❌ 未经验证的绝对化陈述（"完全"、"绝对"、"永远"）
- ❌ 无引用支撑的权威性断言
- ❌ 使用感叹号、省略号等情绪标点
- ❌ 大段文字无分段（超过 10 行）

### 优先原则

- 清晰 > 复杂
- 精确 > 模糊
- 简洁 > 冗长
- 客观 > 主观

## 输出规范

### 标题层级

```markdown
# 一级标题：论文题目
## 二级标题：章节
### 三级标题：小节
#### 四级标题：要点
```

### 段落结构

- **主题句**：段落首句，概括本段核心
- **展开句**：2-4 句，提供论据或细节
- **结论句**：总结或过渡

典型段落长度：3-5 句，80-150 字

### 引用格式

**中文文献**：
- 单作者：`（张三，2023）`
- 双作者：`（张三和李四，2023）`
- 多作者：`（张三等，2023）`

**英文文献**：
- 单作者：`(Smith, 2023)`
- 双作者：`(Smith & Jones, 2023)`
- 多作者：`(Smith et al., 2023)`

### 参考文献格式

```
[1] 作者. 标题[J]. 期刊名, 年份, 卷(期): 页码.
[2] Author. Title[C]//Conference Name. Year: pages.
```

## 检查清单

### 执行前

- [ ] 明确论文的研究问题或论点
- [ ] 确认已有足够的文献支撑
- [ ] 了解目标期刊或会议的格式要求（如有）

### 执行中

- [ ] 每写完一段，检查主题句是否清晰
- [ ] 每引用一个观点，立即添加引用标记
- [ ] 避免连续 3 段以上无小标题分隔

### 执行后

- [ ] 通读全文，检查逻辑连贯性
- [ ] 验证所有引用都在参考文献中
- [ ] 检查术语使用是否一致
- [ ] 确认无口语化、情绪化表达
- [ ] 段落长度适中（无超过 10 行的段落）
- [ ] 标题层级正确（无跳级）

## 示例对比

### ❌ 不符合规范

```
多智能体系统真的非常重要！！！它可以解决很多问题，
比如说分布式计算啊、协作任务啊等等。大家都知道，
这个领域发展得超级快......

最近有学者提出了一些新方法，效果还不错。
```

**问题**：
- 感叹号、省略号
- 口语化（"真的"、"超级"、"还不错"）
- 无引用
- 段落无主题句
- 过于模糊（"一些"、"很多"）

### ✅ 符合规范

```
多智能体系统（Multi-Agent System, MAS）在分布式问题求解
领域发挥关键作用（Wooldridge, 2009）。该系统通过多个
自主智能体的协作，实现单一智能体无法完成的复杂任务
（Ferber, 1999）。近年来，MAS 在协同控制（Smith et al.,
2021）、分布式优化（Zhang & Wang, 2022）等场景中展现
出显著优势。

Wang 等（2023）提出的分层协调机制，在大规模智能体
系统中将通信开销降低了 40%，为该领域提供了新的解决
思路。
```

**优点**：
- 首句定义关键术语
- 每个论点都有引用
- 客观叙述，无情绪化
- 段落结构清晰
- 数据具体（40%）

## 特殊场景处理

### 文献综述

额外要求：
- 按主题组织，而非按文献逐一介绍
- 批判性分析现有研究的不足
- 明确指出研究缺口

### 实证研究

额外要求：
- 方法部分可复现（细节充分）
- 结果部分客观呈现数据
- 讨论部分需与前人研究对比

### 理论构建

额外要求：
- 概念定义严格（必要时使用形式化表达）
- 假设明确列出
- 逻辑推演过程清晰
